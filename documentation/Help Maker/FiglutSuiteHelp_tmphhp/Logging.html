Logging
 Previous Top Next
The following settings can be configured under the Logging category:

Ø   To File : Whether or not to log to a text log file in the executing directory.
Ø   To Windows Event Log : Whether or not to log to the Windows Event Log.
Ø   To Console : Whether or not to write all log messages to the console. Useful when running the service as a console application i.e. running the executable with the /test_mode switch.
Ø   Log File Name : The name of the text log file to log to. The log file is written in the executing directory.
Ø   Event Source Name : The name of the event source to use when logging to the Windows Event Log.
Ø   Event Log Name : The name of the Windows Event Log to log to.
Ø   Logging Level : The extent of messages being logged:
Ø   None = logging is disabled.
Ø   Minimum = logs server start/stop and exceptions.
Ø   Normal = logs additional information messages.
Ø   Maximum = logs all requests and responses to the server.
