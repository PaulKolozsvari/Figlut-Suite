Figlut Web Service Overview
 Previous Top Next
The Figlut Web Service is an out-of-the-box RESTful Web Service that enables the exposure of a database to the web without requiring any development for basic CRUD (Create, Read, Update, Delete) operations.  Supports XML, JSON and CSV media types.

This dynamic access to a database via the Figlut Web Service an be made possible by firstly configuring the Figlut Web Service to point to a database. From that point onwards the Figlut Web Service will manage the resources (tables, fields, columns etc.) in the database. This enables client applications to access and edit resources in the database via the Figlut Web Service. Hence the Figlut Web Service providing a mechanism for exposing the resources of a database to the Internet for client applications to consume without any server-side development required.

For example, if the given database contains a table named "User" containing user data, a client application could then make a HTTP GET request to a URI such as http://www.mydomainname.com/Figlut/User, which would return to the caller all of the records in the 'User" table in either XML, JSON or CSV format. An HTTP POST, PUT or DELETE request could also be made to the same above URI in order to create, update or delete records in the "User" table. More advanced requests are also supported that enable the caller to run filtered queries against the database.
